# 答辩 PPT 大纲

本文档用于直接制作课程设计答辩 PPT。建议控制在 12 到 15 页，重点突出“YOLOv11 迁移训练 + 中文语义映射 + 可演示系统”。

## 第 1 页：封面

标题：

```text
基于 YOLOv11 迁移训练与中文语义映射的多类别手语识别与无障碍交流辅助系统
```

建议内容：

- 课程名称：计算机视觉 / 模式识别课程设计；
- 团队成员、学号、班级；
- 指导教师；
- 日期：2026 年 5 月。

讲稿要点：

本项目围绕手语识别和无障碍交流辅助展开，使用 YOLOv11 完成多类别手语动作检测，并将英文标签转换为中文语义，实现图片、批量图片和摄像头演示。

## 第 2 页：研究背景

建议标题：

```text
研究背景：手语交流辅助需求
```

页面内容：

- 听障群体与健听人群之间存在沟通障碍；
- 手语学习成本较高，普通人难以直接理解；
- 计算机视觉可通过摄像头识别手势，降低沟通成本；
- 课程设计目标是实现可训练、可验证、可演示的系统原型。

讲稿要点：

传统方案要么依赖人工特征，要么只做单张图片分类。本项目希望构建更完整的系统，从检测模型训练到中文语义输出都覆盖。

## 第 3 页：项目目标

页面内容：

| 目标 | 说明 |
| --- | --- |
| 模型训练 | YOLOv11 迁移训练 35 类手语 |
| 多输入识别 | 图片、视频、摄像头、批量图片 |
| 中文理解 | 英文标签映射为中文含义 |
| 无障碍输出 | 支持语音播报和历史记录 |
| 工程演示 | HTML 页面和 PyQt 界面 |

讲稿要点：

项目不是只跑一次推理，而是覆盖课程设计需要的训练、验证、部署和展示流程。

## 第 4 页：技术路线

页面图示：

```text
YOLO 数据集
  -> 数据检查
  -> YOLOv11 迁移训练
  -> 验证集评估
  -> 图片/视频/摄像头推理
  -> 中文语义映射
  -> 语音播报 + 历史记录 + HTML 演示
```

讲稿要点：

YOLOv11 负责视觉识别，中文映射和语音播报负责交互表达，历史记录和 CSV 导出负责结果追溯。

## 第 5 页：数据集说明

页面内容：

| 项目 | 数值 |
| --- | --- |
| 类别数 | 35 |
| 训练图片 | 2148 |
| 验证图片 | 210 |
| 标注格式 | YOLO 检测格式 |
| 输入尺寸 | 640 x 640 |

可展示类别示例：

```text
time, happy, please, friend, thank you, I/me, love, good, what, name, introduce
```

讲稿要点：

数据集采用 YOLO 格式，每张图片有对应标签文件，标签中包含类别和归一化检测框坐标。

## 第 6 页：系统模块

页面内容：

| 模块 | 文件 | 作用 |
| --- | --- | --- |
| 数据检查 | `scripts/check_dataset.py` | 检查标注质量和类别分布 |
| 训练验证 | `train/` | YOLOv11 训练、验证、对比 |
| 检测后端 | `app/detector.py` | 模型加载和推理封装 |
| 中文映射 | `configs/label_map.json` | 英文到中文 |
| 语音历史 | `speech.py`、`recorder.py` | 播报与 CSV 导出 |
| HTML 演示 | `new-shoyuDetection/app.py` | Web 页面展示 |

讲稿要点：

模块化拆分后，命令行、桌面界面和 Web 页面可以复用同一套检测逻辑和映射配置。

模型在第 63 轮早停，最佳权重来自第 33 轮，说明后续继续训练提升有限。

## 第 9 页：验证结果

页面内容：

| Precision | Recall | mAP@0.5 | mAP@0.5:0.95 |
| --- | --- | --- | --- |
| 0.9730 | 0.9831 | 0.9834 | 0.7986 |

讲稿要点：

Precision 和 Recall 都超过 0.97，说明模型在验证集上既能较准确地预测类别，也能较完整地召回目标。mAP@0.5:0.95 接近 0.80，说明定位能力也较稳定。

## 第 10 页：中文语义映射

页面内容：

| 英文标签 | 中文含义 |
| --- | --- |
| `thank you` | 谢谢 |
| `I/me` | 我 |
| `love` | 爱 |
| `good` | 好 |
| `friend` | 朋友 |
| `please` | 请 |

讲稿要点：

YOLO 模型输出的是英文类别标签。为了适配中文无障碍交流场景，项目加入 `label_map.json`，将英文标签转换成中文语义。

## 第 11 页：HTML 演示功能

页面内容：

- 单张图片检测；
- 批量上传图片测试；
- 摄像头实时检测；
- 后台分段识别，减少卡顿；
- 中英文标签、置信度和检测框展示；
- 简单短语拼接；
- 保存检测结果。

讲稿要点：

HTML 页面是答辩推荐入口，批量上传可以快速展示多样本效果，摄像头页面则体现实时检测能力。

## 第 12 页：系统演示安排

页面内容：

演示顺序：

1. 打开 `http://localhost:5000`；
2. 单张图片检测；
3. 批量上传多张测试图片；
4. 摄像头识别 2 到 3 个手势；
5. 展示中文含义和简单短语；
6. 保存检测结果；
7. 展示验证指标 CSV。

讲稿要点：

如果现场摄像头效果不稳定，可以先用批量图片测试证明模型能力，再解释摄像头场景的泛化差异。

## 第 14 页：不足与改进

页面内容：

不足：

- 当前只支持 35 类；
- 摄像头真实场景会受背景和光照影响；
- 简单短语拼接不等同完整语义翻译；
- YOLOv11n 模型较小，复杂手势容易混淆。

改进：

- 增加真实摄像头数据；
- 尝试 YOLOv11s 或 YOLOv11m；
- 引入手部关键点或时序模型；
- 增强中文句子生成能力。

讲稿要点：

项目已经完成课程设计的核心闭环，后续可以向连续手语理解和真实应用部署方向扩展。

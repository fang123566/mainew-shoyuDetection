# 运行说明

本文档提供项目从数据检查、模型验证、HTML 演示、命令行检测到重新训练的完整运行流程。

## 1. 运行前检查

进入仓库根目录：

```bash
cd /mnt/e/python/CVandPR/mainew-shoyuDetection
```

激活环境：

```bash
source ~/anaconda3/etc/profile.d/conda.sh
conda activate ai_trading
export LD_LIBRARY_PATH=/usr/lib/wsl/lib:$LD_LIBRARY_PATH
```

确认模型权重存在：

```bash
ls weights/yolov11_best.pt
```

确认数据集配置存在：

```bash
ls datasets/sign_language/data.yaml
```

## 2. 数据集检查

训练或写报告前建议先运行：

```bash
python scripts/check_dataset.py --data datasets/sign_language/data.yaml
```

脚本会检查：

- `data.yaml` 是否可读取；
- train、val、test 路径是否存在；
- 图片和标签是否一一对应；
- YOLO 标签是否为 5 列；
- class_id 是否越界；
- 坐标是否在 0 到 1 之间；
- 是否存在空标签、缺失标签、损坏图片；
- 各类别样本数量。

输出：

```text
outputs/csv/dataset_summary.csv
outputs/figures/class_distribution.png
```

## 3. 推荐演示：HTML 页面

当前最推荐的答辩演示入口是 HTML 页面。

启动：

```bash
python new-shoyuDetection/app.py
```

浏览器打开：

```text
http://localhost:5000
```

页面功能：

| 功能 | 操作 |
| --- | --- |
| 图片检测 | 点击“图片检测”，上传单张图片，点击开始检测 |
| 批量测试 | 点击“批量测试”，一次选择多张图片，点击开始批量测试 |
| 摄像头检测 | 点击“摄像头检测”，允许浏览器访问摄像头 |
| 阈值调节 | 调整置信度和 IoU 滑块 |
| 保存结果 | 点击保存结果，图片保存到 `outputs/images/` |

摄像头检测说明：

- 前端画面连续渲染；
- 后台每隔约 1.2 秒抽帧识别；
- 这样可以避免每一帧都发给后端导致页面卡顿；
- 简单短语会根据最近稳定识别结果拼接。

停止服务：

```bash
ss -ltnp | grep ':5000'
kill <pid>
```

## 6. 命令行摄像头检测

```bash
python app/main.py \
  --weights weights/yolov11_best.pt \
  --source 0 \
  --conf 0.5 \
  --speak \
  --save
```

参数说明：

| 参数 | 说明 |
| --- | --- |
| `--source 0` | 使用默认摄像头 |
| `--conf 0.5` | 置信度阈值，演示效果不好可降到 0.25 到 0.40 |
| `--iou 0.45` | NMS IoU 阈值 |
| `--speak` | 开启中文语音播报 |
| `--save` | 保存检测视频 |
| `--top1` | 只保留最高置信度检测结果 |

按 `q` 退出。退出后自动保存历史记录：

```text
outputs/csv/recognition_history.csv
```

## 9. 重新训练 YOLOv11

团队成员通常不需要重新训练。如确实需要复现实验，在 Ubuntu CUDA 环境中运行：

```bash
python train/train_yolov11.py \
  --data datasets/sign_language/data.yaml \
  --model yolo11n.pt \
  --epochs 100 \
  --imgsz 640 \
  --batch 16 \
  --device 0 \
  --workers 4 \
  --name yolov11_sign_language_cuda \
  --patience 30
```

训练输出：

```text
runs/sign_language/yolov11_sign_language_cuda/
```

如果训练出新的最佳模型，需要更新默认权重：

```bash
cp runs/sign_language/yolov11_sign_language_cuda/weights/best.pt weights/yolov11_best.pt
```

## 11. 答辩演示流程建议

建议提前准备 5 张验证集图片或自己拍摄的手势图片，然后按以下顺序演示：

1. 打开 HTML 页面；
2. 展示项目标题和模型设备信息；
3. 单张图片检测，说明检测框、中英文标签和置信度；
4. 批量上传多张图片，说明系统可进行批量测试；
5. 打开摄像头，比出 2 到 3 个训练集包含的手势；
6. 展示简单短语区域；
7. 点击保存结果，展示输出路径；
8. 切换到 PPT 中的训练指标表，说明模型效果。

## 12. 识别效果调参建议

如果现场识别效果不好，按以下顺序调整：

1. 将置信度从 0.50 降到 0.35；
2. 保持手势在画面中央；
3. 让手势停留 1 到 2 秒；
4. 避免复杂背景和背光；
5. 尽量使用训练集中已有的 35 类手势；
6. 如果还不稳定，用批量上传图片演示模型能力，再说明真实摄像头泛化问题。

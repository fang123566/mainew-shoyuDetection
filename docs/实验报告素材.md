## 一、项目背景

手语是听障人士与健听人群之间重要的交流方式。传统手语识别依赖人工提取特征或单一分类模型，存在泛化能力差、实时性不足等问题。本项目基于 YOLO（You Only Look Once）系列目标检测算法，对多类别中国手语动作词进行迁移训练，实现从英文类别标签到中文语义的实时映射，构建一个完整的无障碍交流辅助演示系统。

本项目具有以下课程设计特色：
- **从 YOLOv8 到 YOLOv11 的版本迁移训练**，对比两代模型的精度与速度。
- **多类别目标检测**（35 类），而非简单的图像分类。
- **中文语义映射**，使机器视觉结果能够被中文用户直接理解。
- **语音播报 + 历史记录 + CSV 导出**，形成完整的数据链路。
- **命令行 + PyQt 图形界面双入口**，适合答辩演示。

## 二、技术路线

```
数据准备
    │
    ▼
YOLO 格式标注 ──► 数据集质量检查（check_dataset.py）
    │
    ▼
YOLOv11 迁移训练 ──► 验证集评估 ──► 最佳权重 best.pt
    │
    ▼
YOLOv8 对比实验 ──► 精度/速度/FPS 对比
    │
    ▼
推理部署
    ├── 图片检测（predict_image.py）
    ├── 视频检测（predict_video.py）
    ├── 摄像头实时检测（main.py）
    └── PyQt 图形界面（ui_qt.py）
            │
            ▼
    中文语义映射（translator.py）
            │
            ▼
    语音播报（speech.py） + 历史记录（recorder.py）
```

## 三、系统模块

| 模块 | 文件 | 功能描述 |
| --- | --- | --- |
| 数据检查 | `scripts/check_dataset.py` | 校验 YOLO 格式、类别分布、图片完整性 |
| 模型训练 | `train/train_yolov11.py` | YOLOv11 迁移训练，支持早停和 GPU/CPU |
| 模型验证 | `train/val_yolov11.py` | 评估 Precision、Recall、mAP@0.5、mAP@0.5:0.95 |
| 模型对比 | `train/compare_models.py` | YOLOv8 vs YOLOv11 横向实验 |

| 图形界面 | `app/ui_qt.py` | PyQt5 界面，集成所有功能 |

## 四、数据集说明

### 数据集来源

本项目使用手语词检测数据集，包含 35 个常见手语动作词。

### 数据格式

符合 YOLO 检测标准格式：

```
class_id x_center y_center width height
```

其中所有坐标值均为归一化值（0~1）。

## 五、模型训练参数

### YOLOv11 训练配置

| 参数 | 值 | 说明 |
| --- | --- | --- |
| 预训练模型 | yolo11n.pt | Ultralytics 官方预训练权重 |
| 输入尺寸 | 640×640 | 训练时统一resize |
| 训练轮数 | 100 | 课程设计建议不少于 50 轮 |
| Batch Size | 16 | GPU 显存不足时可降至 8 或 4 |
| 优化器 | SGD（默认） | ultralytics 默认 |
| 学习率 | 自动调度 | lr0=0.01, lrf=0.01 |
| 早停 | patience=30 | 验证集指标 30 轮不提升则停止 |
| 数据增强 | 默认 | Mosaic、Flip、HSV 等 |
| 设备 | 0（GPU）/cpu | 自动检测 |

### 训练输出

```
runs/sign_language/<name>/
├── weights/
│   ├── best.pt    # 验证集最优权重
│   └── last.pt    # 最后一轮权重
├── args.yaml      # 训练参数记录
├── results.csv    # 训练过程指标
└── results.png    # 训练曲线
```

## 七、创新点

1. **YOLOv11 迁移训练框架**：在 YOLO 官方 ultralytics 框架上进行完整的训练-验证-对比流水线，而非仅使用预训练模型做推理。

2. **中文语义映射层**：检测结果从英文标签映射为中文语义，并结合语音播报，形成完整的"检测→理解→播报"无障碍链路。

3. **识别历史与数据导出**：通过 `recorder.py` 模块将每次识别记录持久化为 CSV，便于后续数据分析和报告撰写。

4. **YOLOv8/YOLOv11 对比实验**：在相同数据集上对比两代模型，体现课程设计中的"实验对比"思维。

5. **模块化架构**：检测后端（detector.py）、中文映射（translator.py）、语音（speech.py）、历史（recorder.py）均为独立模块，CLI 和 GUI 共享，降低维护成本。

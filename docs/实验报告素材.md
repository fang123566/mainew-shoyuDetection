# 实验报告素材

本文档内容可直接整理为课程设计报告，也可拆分为答辩 PPT 页面。建议报告标题使用：

《基于 YOLOv11 迁移训练与中文语义映射的多类别手语识别与无障碍交流辅助系统》

## 摘要

针对听障群体与健听人群之间的沟通辅助需求，本文设计并实现了一套基于 YOLOv11 的多类别手语识别系统。系统以 35 类手语动作词为识别对象，采用 Ultralytics YOLOv11 进行迁移训练，将手语识别建模为目标检测任务。在模型推理后，系统通过中文语义映射模块将英文类别标签转换为中文含义，并进一步结合语音播报、识别历史记录、CSV 导出和 HTML 可视化界面，形成面向课程答辩演示的无障碍交流辅助原型。实验在 Ubuntu WSL2 + CUDA 环境中完成，使用 `yolo11n.pt` 预训练权重训练 63 轮，最佳模型在验证集上取得 Precision 0.9730、Recall 0.9831、mAP@0.5 0.9834、mAP@0.5:0.95 0.7986 的结果。系统支持图片检测、批量图片测试、视频检测和摄像头实时检测，具有较好的完整性和可演示性。

关键词：手语识别；YOLOv11；目标检测；迁移学习；中文语义映射；无障碍交流

## 1. 研究背景与意义

手语是听障人士表达信息的重要方式，但多数健听人群并不掌握手语，导致双方在日常交流中存在明显障碍。随着计算机视觉和深度学习的发展，利用摄像头自动识别手语动作成为一种可行的辅助交流方案。

传统手语识别方法通常依赖人工设计特征，如肤色分割、轮廓特征、手部关键点或模板匹配。这类方法对背景、光照和姿态变化较敏感，泛化能力有限。深度学习方法能够自动学习图像特征，但如果只使用普通图像分类模型，往往无法明确定位手部区域，也不利于在复杂场景中展示检测过程。

本项目选择 YOLOv11 目标检测模型，将手语动作作为检测目标进行训练。这样不仅可以输出类别，还可以输出手部动作所在位置，便于在界面中绘制检测框，提高演示可解释性。同时，项目将英文检测标签映射为中文语义，并增加语音播报和历史记录，形成更贴近无障碍交流场景的系统。

## 2. 项目目标

本项目的目标包括：

- 完成 YOLOv11 在手语数据集上的迁移训练；
- 构建 35 类手语动作词识别模型；
- 支持图片、视频、摄像头和批量图片测试；
- 实现英文标签到中文语义的映射；
- 实现简单短语拼接和中文语音播报；
- 保存识别历史并支持 CSV 导出；
- 提供 HTML 页面和 PyQt 界面，便于课程答辩演示；
- 生成训练记录、验证指标和报告素材。

系统整体技术路线如下：

系统输入可以是单张图片、视频文件、摄像头画面或多张图片。YOLOv11 模型输出检测框、类别和置信度后，系统将结果整理为结构化数据，再交给中文映射、语音播报、历史记录和界面展示模块使用。

## 4. 数据集说明

### 4.1 数据格式

数据集采用标准 YOLO 检测格式，每张图片对应一个同名 `.txt` 标签文件，标签格式为：

```text
class_id x_center y_center width height
```

其中：

- `class_id` 表示类别编号；
- `x_center` 和 `y_center` 表示检测框中心点；
- `width` 和 `height` 表示检测框宽高；
- 坐标值均为 0 到 1 的归一化值。

### 4.2 数据集配置

配置文件：

```text
datasets/sign_language/data.yaml
```

主要内容：

```yaml
path: .
train: new-shoyuDetection/datasets/images/train
val: new-shoyuDetection/datasets/images/val
test: new-shoyuDetection/datasets/images/val
nc: 35
```

## 6. YOLOv11 训练过程

### 6.1 训练环境

| 项目 | 配置 |
| --- | --- |
| 系统 | Ubuntu WSL2 |
| Conda 环境 | `ai_trading` |
| Python | 3.10.20 |
| PyTorch | 2.0.0+cu117 |
| CUDA | 11.7 |
| GPU | NVIDIA GeForce RTX 4060 Laptop GPU |
| Ultralytics | 8.4.46 |

### 6.2 训练命令

```bash
python train/train_yolov11.py \
  --data datasets/sign_language/data.yaml \
  --model yolo11n.pt \
  --epochs 100 \
  --imgsz 640 \
  --batch 16 \
  --device 0 \
  --workers 4 \
  --name yolov11_sign_language_cuda \
  --patience 30
```

### 6.3 参数设置

| 参数 | 值 | 说明 |
| --- | --- | --- |
| 模型 | `yolo11n.pt` | YOLOv11 nano 预训练权重 |
| 输入尺寸 | 640 | 图像输入分辨率 |
| Epochs | 100 | 最大训练轮数 |
| Batch Size | 16 | 批量大小 |
| Device | 0 | 使用第 0 块 GPU |
| Workers | 4 | 数据加载线程 |
| Patience | 30 | 早停耐心值 |
| AMP | true | 混合精度训练 |

### 6.4 训练结果

训练最大轮数为 100，实际训练到第 63 轮触发 EarlyStopping，最佳模型出现在第 33 轮。

## 7. 实验结果分析

Precision 达到 0.9730，说明模型预测出的手语目标中绝大部分为正确类别。Recall 达到 0.9831，说明验证集中的绝大多数手语目标都能被成功召回。mAP@0.5 达到 0.9834，说明在常用检测评价标准下模型具有较高的整体检测精度。mAP@0.5:0.95 为 0.7986，说明在更严格的定位要求下，模型仍保持了较好的边界框定位能力。

从训练过程看，模型在前几轮指标提升很快，说明 YOLOv11 预训练权重在手语检测任务上具有较好的迁移能力。后期 mAP@0.5:0.95 在 0.79 到 0.80 附近波动，说明模型逐渐收敛，继续训练提升有限，因此 EarlyStopping 能有效节省训练时间。

真实摄像头演示效果可能低于验证集指标，主要原因包括背景差异、光照变化、手部尺度变化和动态手势模糊。因此系统在 HTML 演示中加入后台分段识别，降低页面卡顿，同时建议答辩时适当降低置信度阈值并保持手势稳定。

## 8. 系统演示功能

### 8.1 图片检测

用户上传单张图片后，系统返回带检测框的结果图，并显示英文标签、中文含义和置信度。

### 8.2 批量上传测试

用户可一次上传多张图片，系统逐张检测并返回缩略图和表格结果。该功能适合答辩时快速展示多样本检测能力。

### 8.3 摄像头实时检测

系统调用浏览器摄像头，前端持续显示画面，后端按时间片抽帧识别。这样可以避免每一帧都请求后端造成卡顿。

### 8.4 中文语义映射与短语拼接

模型输出英文标签后，系统通过 `configs/label_map.json` 映射为中文。例如 `thank you` 映射为“谢谢”，`I/me` 映射为“我”，`love` 映射为“爱”。连续识别结果会拼接成简单短语。

### 8.5 历史记录与导出

系统可以记录每次识别的时间、英文标签、中文含义、置信度和检测框坐标，并导出为 CSV，便于后续分析和报告截图。

## 9. 创新点

1. 使用 YOLOv11 完成手语动作检测迁移训练，而不是仅调用现成模型；
2. 将英文检测标签映射为中文语义，提高中文场景可用性；
3. 加入语音播报和历史记录，形成“检测、理解、播报、存档”的完整链路；
4. 支持批量图片测试，便于模型效果验证和答辩展示；
5. 摄像头演示采用后台分段识别，兼顾实时画面流畅度和检测结果展示；
6. 训练、验证、数据检查、报告素材和演示界面完整，符合课程设计工程化要求。

## 10. 不足与展望

当前系统仍存在一些不足：

- 识别类别数量有限，仅包含当前数据集的 35 类；
- 简单短语拼接基于规则，没有使用真正的自然语言模型；
- YOLOv11n 模型较轻量，复杂场景下可能出现误检；
- 摄像头场景与训练集存在差异，泛化能力仍需提升；
- 当前系统主要识别静态或短时手势，对连续动态手语理解有限。

后续可改进方向：

- 采集真实摄像头数据重新标注并微调；
- 尝试 YOLOv11s 或 YOLOv11m 提升复杂手势识别能力；
- 引入 MediaPipe 或关键点模型辅助建模；
- 使用时序模型处理连续手语片段；
- 增加更完整的中文句子生成和语音交互功能。

## 11. 报告结论

本文完成了一套基于 YOLOv11 的多类别手语识别与无障碍交流辅助系统。系统以 35 类手语动作为检测目标，完成了数据集检查、模型迁移训练、验证评估、中文语义映射、语音播报、历史记录、批量测试和实时演示等功能。实验结果表明，训练得到的 YOLOv11 模型在验证集上取得 Precision 0.9730、Recall 0.9831、mAP@0.5 0.9834、mAP@0.5:0.95 0.7986 的效果，能够满足课程设计中的模型训练和系统演示要求。项目不仅体现了目标检测算法在手语识别中的应用，也展示了从模型训练到交互式应用落地的完整工程流程。

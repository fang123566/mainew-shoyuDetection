# 项目说明

## 1. 项目名称

基于 YOLOv11 迁移训练与中文语义映射的多类别手语识别与无障碍交流辅助系统

## 2. 项目定位

本项目是计算机视觉课程设计项目，目标不是简单调用一个目标检测模型，而是构建一个从数据检查、模型训练、模型验证、实时推理、中文语义映射、语音播报、历史记录到可视化演示的完整系统。

项目面向的典型场景是：用户通过图片、视频或摄像头展示手语动作，系统识别手势类别，将英文检测标签映射为中文语义，并在界面上展示结果，必要时通过语音播报辅助交流。

## 3. 建设目标

项目最终需要体现以下能力：

- YOLOv11 迁移训练；
- 35 类手语动作词检测；
- 图片、视频、摄像头多输入源推理；
- 英文标签到中文语义映射；
- 简单短语拼接；
- 中文语音播报；
- 识别历史记录与 CSV 导出；
- HTML 页面演示和 PyQt 桌面界面演示；
- 数据集检查、验证指标和报告图表输出；
- 团队协作可读的代码结构和文档。

```bash
conda activate ai_trading

```

浏览器打开：

```text

```

演示顺序建议：

1. 单张图片检测，展示检测框和中英文标签；
2. 批量上传图片，展示多样本测试能力；
3. 摄像头实时检测，展示系统实时性；
4. 连续比出几个手势，展示简单短语拼接；
5. 保存检测结果，说明输出可追溯。

## 8. 项目特色

1. 从 YOLOv11 迁移训练到部署演示形成完整链路；
2. 检测结果不是停留在英文类别，而是映射到中文语义；
3. 加入语音播报和历史记录，更贴近无障碍交流辅助系统；
4. 支持批量图片测试，便于课程答辩快速展示模型效果；
5. 摄像头检测采用后台分段识别，减少前端画面卡顿；
6. 文档、训练记录和报告素材完整，便于团队成员接手。

## 9. 不足与后续改进

当前项目仍存在以下限制：

- 模型只训练了 35 个静态或短时手语动作词，不是真正完整连续手语翻译；
- 摄像头效果受背景、光照、手部大小和运动模糊影响较大；
- YOLOv11n 模型较小，实时性好，但复杂手势区分能力有限；
- 简单短语拼接主要基于最近识别结果，没有引入时序语言模型。

后续可改进方向：

- 采集真实摄像头场景数据并重新标注微调；
- 尝试 YOLOv11s 或 YOLOv11m 提升识别能力；
- 引入关键点检测或时序模型处理连续手语；
- 增加更丰富的中文短语规则和语音交互逻辑；
- 将演示页面封装为更完整的 Web 应用。

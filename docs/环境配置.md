# 环境配置

## 推荐环境

| 项目 | 推荐配置 |
| --- | --- |
| 操作系统 | Windows 11 + WSL2 Ubuntu，或原生 Ubuntu |
| Conda 环境 | `ai_trading` |
| Python | 3.10.x |
| PyTorch | 2.0.0+cu117 或兼容版本 |
| Ultralytics | 8.4.46 |
| GPU | NVIDIA GPU 可选，CPU 可运行演示但较慢 |

当前项目在 WSL2 + NVIDIA GeForce RTX 4060 Laptop GPU 环境验证通过。

## 激活环境

```bash
source ~/anaconda3/etc/profile.d/conda.sh
conda activate ai_trading
export LD_LIBRARY_PATH=/usr/lib/wsl/lib:$LD_LIBRARY_PATH
```

## 依赖安装参考

如果需要重建环境，可参考：

```bash
conda create -n ai_trading python=3.10 -y
conda activate ai_trading
pip install -r requirements.txt
pip install seaborn matplotlib
```

GPU 环境需要先安装与本机 CUDA 匹配的 PyTorch。课程演示只要求能启动 Web 和加载 `weights/yolov11_best.pt`。

## 必要文件

| 文件 | 作用 |
| --- | --- |
| `weights/yolov11_best.pt` | Web 默认模型权重，保留在仓库中 |
| `configs/label_map.json` | 英文类别到中文语义的映射 |
| `configs/report_summary.json` | 评估页回退摘要 |
| `new-shoyuDetection/static/analysis/` | 已提交的真实评估图表 |
| `datasets/sign_language/data.yaml` | 数据集配置 |

## 常见问题

- 找不到 `ultralytics`：确认已激活 `ai_trading`，或重新安装 requirements。
- WSL 中 GPU 不可用：确认 NVIDIA 驱动、WSL CUDA 支持和 `LD_LIBRARY_PATH`。
- 摄像头打不开：检查浏览器权限；WSL/远程环境可能无法直接访问 Windows 摄像头，可在 Windows 浏览器中访问 Web 服务。
- 评估页没有图表：确认 `new-shoyuDetection/static/analysis/` 下存在 PNG 文件，或重新运行 `scripts/generate_analysis.py`。
